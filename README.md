# AgendaFácil — MVP Sprint 1

Repositório de entrega do MVP da disciplina de Engenharia de Requisitos (PUC-Rio, Especialização em Engenharia de Software).

## Sobre o projeto

O **AgendaFácil** nasceu de um problema real: donos de salões, barbearias e estúdios de unha perdem tempo demais atendendo WhatsApp e telefone só pra marcar horário. A ideia é simples — um app web onde o cliente final escolhe serviço, profissional e horário livre, e agenda sozinho pelo celular. Do outro lado, o dono do negócio tem um painel limpo pra ver a agenda do dia e configurar tudo.

Neste repositório está a documentação completa do MVP: desde a concepção (Lean Inception) até o protótipo das telas da primeira sprint.

### Wireframes (protótipo de baixa fidelidade)

Os wireframes foram feitos no Figma, em estilo low-fi (tons de cinza, sem cores, mobile-first 360×800px). São 9 telas que cobrem toda a Sprint 1:

| # | Tela | História |
|---|------|----------|
| 01 | Cadastro do negócio | US-01 |
| 02 | Painel / Agenda do dia | US-05 |
| 03 | Serviços | US-02 |
| 04 | Horário de funcionamento | US-02 |
| 05 | Profissionais | US-02 |
| 06 | Página pública — escolha de serviço | US-03 |
| 07 | Página pública — profissional + horários | US-03 |
| 08 | Página pública — confirmação | US-04 |
| 09 | Tela de sucesso | US-04 |

Os dois fluxos principais:
- **Dona do negócio:** Cadastro → Painel ↔ Serviços / Horários / Profissionais
- **Cliente final:** Escolhe serviço → Escolhe profissional e horário → Confirma → Sucesso

→ pasta [`wireframes/`](wireframes/)
